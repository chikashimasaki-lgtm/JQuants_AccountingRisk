# 東証プライム 会計リスク・スクリーナー（J-Quants API）

無料の J-Quants API（約12週遅延）で取得できるサマリー財務から「利益の質」を評価し、
会計リスク（利益操作・急悪化）の兆候がある東証プライム銘柄をランク付けする GAS アプリ。
**投資助言ではなく、スクリーニング／監査教育目的。**

## スコア設計（すべて無料データで計算可能）

| 指標 | 式 | 意味 |
|---|---|---|
| **アクルーアル比率（主指標）** | (当期純利益 − 営業CF) / 総資産 | 利益が現金の裏付けを欠く＝利益操作の代表的赤信号 |
| 黒字なのに営業CFマイナス | 純利益>0 かつ 営業CF<0 | 現金を伴わない利益 |
| 営業利益率の急変 | 当期 − 前期の営業利益率 | マージン粉飾・一過性計上 |
| 自己資本比率 | 純資産 / 総資産 | 財務体質の劣化 |
| 特別損益依存 | (経常利益 − 純利益) / 売上 | 特別項目での利益調整 |

主指標（アクルーアル）を母集団で偏差値化し、フラグで軽く加点して **会計リスク偏差値** としてランキング。

> ⚠️ 本家 Beneish M-Score は売掛金・売上原価・減価償却が必須だが、無料 `/fins/statements` では取得不可。
> そのため取得可能な項目（利益・営業CF・総資産等）で計算できるアクルーアル系を主軸にしている。

## セットアップ

1. [J-Quants](https://jpx-jquants.com/) に登録（無料Freeプラン）
2. Apps Script プロジェクトを作成し、`src/` の中身を push（clasp）またはコピペ
3. スクリプトプロパティに認証情報を設定（どちらか）
   - `JQUANTS_MAIL` と `JQUANTS_PASSWORD`
   - または `JQUANTS_REFRESH_TOKEN`（マイページで発行したもの）
4. スプレッドシートを再読み込み → メニュー「会計リスク」

## 使い方

1. **セットアップ（シート作成）**
2. **① プライム銘柄を取得** … `/listed/info` からプライム銘柄マスタを作成
3. **② 財務データを収集/続行** … 銘柄ごとに `/fins/statements` の全履歴を取得（時間内で分割実行し、90秒後に自動再開）
4. **③ リスクスコアを計算** … 通期(FY)決算から指標を算出しランキング化
5. **JSON出力** … GitHub Pages 表示用に `accounting_risk_prime.json` を Drive に出力

## 注意・制約

- 無料枠は **約12週遅延** ＋ 履歴範囲・レート制限あり。決算分析用途なので遅延は問題にならない。
- 四半期は営業CF未開示が多いため、**通期(FY)決算を主対象**にしている。
- コード内の `要確認` コメント（MarketCode 名称・項目名）は J-Quants 仕様変更で変わり得る。
